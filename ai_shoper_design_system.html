Кнопки

4 типа: primary, secondary, ghost, danger. Высота по умолчанию 44px (правило Apple HIG для tap-targets).

Типы
Размеры
Состояния
Полная ширина (для CTA)
Spec
Высота 44px = минимум для tap-targets (Apple HIG). Большая 52px для главных CTA. Hover: лёгкое translateY(-1px) + смена цвета. Active: возврат + затемнение. Disabled: opacity: 0.4 + cursor: not-allowed. Иконки в кнопках всегда 18px, gap 8px.

Поля ввода

Голосовой ввод — первичный (большая зелёная кнопка). Текст — вторичный, но всегда доступен.

Главное поле (с голосом)
робот-пылесос до 30к
наушники до 15к
куртка зима M
Spec
Высота поля 52px (с голосом). Радиус --r-lg. Focus: 3px glow цвета акцента. Иконка слева 20px, отступ 16px. Голосовая кнопка 40px, отступ от края 6px. Подсказки-чипсы под полем — клик заполняет поле.
06 / 10

Бейджи

Тонкие индикаторы статуса и контекста. 6 семантических цветов + нейтральный.

Лучшая цена Проверен Подорожал Накрутка Сезонная скидка Без отзывов −15% Premium
Spec
Радиус: --r-full (pill-форма). Fill: 15% opacity от цвета, text: чистый цвет. Не использовать как кнопки — только индикаторы. Не больше 1-2 на карточку.

Состояния загрузки

Это критичный экран — юзер ждёт 30-60 секунд. Не общий spinner, а конкретные этапы поиска.

Этапы поиска (главное)

Выбираю топ-3
Skeleton (для списков)
Spec
Этапы переходят: pending → active (с спиннером) → done (с галочкой и зелёной подложкой). Длительность каждого этапа должна примерно соответствовать реальному времени API-запроса. Skeleton: shimmer-анимация 1.5s ease-in-out infinite.
08 / 10

Empty states

Пустые состояния — это не "ничего нет", а призыв к действию. Иконка + короткий текст + кнопка.

Пока ничего не отслеживаешь
Когда найдёшь интересный товар, нажми «Следить» — пришлём алерт когда подешевеет
Spec
Иконка 64×64 в круге --bg-raised, текст по центру, max-width описания 260px. Главный CTA — primary кнопка. Без декоративных иллюстраций.

Bottom sheet

Для уточнения запроса, выбора опций, шеринга. Не модалка — она перегружает в TG Mini App.

Главный экран в фоне

Spec
Bottom sheet всегда снизу — лучше для большого пальца на телефоне. Handle сверху — индикатор "можно свайпнуть". Background dim'ится при открытии. Не использовать для критичных подтверждений (используй modal на весь экран).
